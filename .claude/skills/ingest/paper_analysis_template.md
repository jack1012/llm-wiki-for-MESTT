# 論文深度分析模板 (5-Pass Framework)

本模板規定了 ingest 技能中論文類資源（`.pdf`）的標準分析流程與輸出格式。

---

## Frontmatter 標準格式

```yaml
---
title: "summary-{檔案slug}"
type: source
tags: [數學教育, 轉移, 課程等相關標籤]
sources: [raw/02-papers/論文完整檔名.pdf]
last_updated: YYYY-MM-DD
---
```

**命名規則：**
- `title`: 格式 `"summary-{slug}"`，slug 由論文標題的關鍵詞組成（kebab-case）
- `tags`: 論文的主題分類（中文標籤，多個用逗號分隔）
- `sources`: 指向原始 PDF 檔案的相對路徑（不要指向 archive）
- `last_updated`: 分析完成日期（YYYY-MM-DD）

---

## 內容結構：5-Pass 深度分析

### **Pass 1：文獻基本資料整理**

提取論文的元資訊和核心屬性。此 Pass 的目的是建立快速參考。

```markdown
## 📝 第一步：文獻基本資料整理 (Pass 1)

- **文章標題 (原文)**：[英文原標題]
- **發表期刊/出版社名稱**：[期刊名稱或出版社]
- **作者**：[作者名單]
- **發表年份**：[YYYY]
- **關鍵字 (Keywords)**：[[概念1]], [[概念2]], [[概念3]] （使用雙向連結）
- **檔案連結**：[[相對路徑/檔案名|顯示文本]]
- **研究主題**：[1-2句話總結]
- **問題意識**：[研究背景與動機，2-3句話]
- **研究問題**：[核心研究問題，通常是"X 如何影響 Y"]
- **研究方法**：[定性/定量/混合研究，樣本量等]
- **研究結果**：[核心發現摘要]
- **研究討論**：[論文討論的理論含義]
- **未來建議**：[論文提出的未來研究方向或實踐建議]
```

---

### **Pass 2：文章深度分析**

進行 5W1H 式的結構化分析，回答"它在說什麼"的問題。

```markdown
## 🔍 第二步：文章深度分析 (Pass 2)

1. **它的問題意識是什麼？**
   [為什麼這個研究很重要？背景脈絡是什麼？]

2. **它討論的問題是什麼？**
   [論文的核心研究問題或假設是什麼？]

3. **它採取的研究方法是什麼？**
   [研究設計、樣本、資料收集方式、分析方法]

4. **它得到的結論是什麼？**
   [主要發現、統計結果、質性發現摘要]

5. **它對這個研究的建議是什麼？**
   [論文作者針對教育/理論/未來研究的建議]
```

---

### **Pass 3：單篇期刊論文評論**

撰寫該論文的綜合評論，包括引用格式、貢獻度評估、缺陷指出和核心金句。

```markdown
## ✒️ 第三步：單篇期刊論文評論 (Pass 3)

**【APA 7th 格式出處】**：
[完整 APA 引用]

1. **這篇論文在說什麼？**
   [1段話的核心敘述]

2. **它有什麼貢獻？我們從它學到什麼？**
   [該論文的原創性和實踐意義]

3. **它有什麼缺陷或不足的地方？**
   [樣本限制、方法論弱點、普遍性問題等]

4. **它的研究發現與討論推論 (核心金句)**：
   "[直接引用或改寫的核心觀點]"
```

---

### **Pass 4：理論模型與關鍵參數**

將論文的理論框架視覺化，並提取可應用的變項映射。

```markdown
## 📊 第四步：理論模型與關鍵參數 (Pass 4)

### 1. 理論錨點 (Theoretical Anchors)
- **核心來源**：[該論文基於的主要理論/框架]
- **支撐理論**：[補充性理論]
- **對話文獻**：[相關的對標研究]

### 2. 變項映射 (Variable Mapping)
- **自變項**：[獨立變數，通常是原因]
- **中介變項**（如有）：[介於原因和結果之間的機制]
- **控制變項**：[被控制的背景因素]
- **依變項**：[因變數，即研究的結果]

### 3. 模型視覺化 (Mermaid Model)
\`\`\`mermaid
graph LR
    A[變項1] -->|關係| B[變項2]
    C[變項3] -->|影響| B
\`\`\`
```

---

### **Pass 5：知識小卡合成點**

提煉論文中最可複用的知識點，每個卡片對應一個原子概念。每張卡片應該：
- **標題清晰**：一個可獨立理解的知識點或概念
- **可複用**：足夠通用，可應用到其他研究或實踐中
- **含有反向連結**：指向該卡片來自哪篇論文（Pass 5 卡片存在 `wiki/cards/` 中）

```markdown
## 💡 第五步：知識小卡合成點 (Atomic Cards)

- **Card 1**：[[概念名稱]] - [該卡片的核心觀點，1-2句話]
- **Card 2**：[[另一概念]] - [該卡片的核心觀點，1-2句話]
- **Card N**：[[第N個概念]] - [該卡片的核心觀點，1-2句話]

**說明**：
- 每張卡片應作為獨立的 `wiki/cards/` 檔案存在
- 卡片命名：`{概念名稱}.card.md`（例：`criterial-reference-in-math-education.card.md`）
- 卡片應包含反向連結指向源論文摘要頁
```

論文分析完成後，可在文末追加讀者活動紀錄（註解形式）：

```markdown
## 📜 讀者活動紀錄 (Activity Logs)

- %% YYYY-MM-DD: [操作者名稱]: [操作簡述。例："使用本模板完成 Ingest，檔案已歸檔至 raw/09-archive/。"] %%
```

## 輸出檢查清單

完成 5-Pass 分析後，確保：

- [ ] Frontmatter 中 `title`, `type`, `tags`, `sources`, `last_updated` 均已填寫
- [ ] Pass 1 提取的關鍵字都是雙向連結 `[[]]` 格式
- [ ] Pass 2 的 5 個問題均已回答，句子清晰完整
- [ ] Pass 3 包含完整的 APA 引用格式
- [ ] Pass 4 包含理論錨點、變項映射和（如適用）Mermaid 圖表
- [ ] Pass 5 至少包含 3-5 張原子卡片
- [ ] 所有雙向連結指向既有或將要建立的頁面（避免死連結）
- [ ] 文末包含活動日誌記錄（可選）

## 卡片 (Card) 格式規範

### 核心原則

**一張卡片 = 一個洞察**
- 每張卡片聚焦**單一、明確的主題**
- 篇幅：150–200 字（中文）或 30–50 詞（英文概念名）
- 目標：濃縮文獻、數據或思考的核心結論
- 效果：易於引用、組合和跨領域應用

### 卡片格式

每張原子卡片應存放在 `wiki/cards/` 中，格式如下：

```yaml
---
title: "卡片標題（一個洞察的名稱）"
type: card
tags: [主題標籤1, 主題標籤2]
sources: [wiki/sources/summary-來源-slug.md]
last_updated: YYYY-MM-DD
---

## 洞察（核心內容，150–200字）

[該卡片表達的核心觀點、發現或機制。直擊要點，避免冗長解釋。]

## 應用場景（可選，1–2個具體例子）

[該洞察可應用的1–2個具體情景。簡潔為主。]

## 相關卡片

- [[相關卡片1.card]] — 說明關聯
- [[相關卡片2.card]] — 說明關聯
```

### 卡片設計原則

| 原則 | 說明 | ❌ 反面例子 | ✅ 正面例子 |
|------|------|----------|----------|
| **一洞察** | 每張卡片表達單一觀點 | 「教育改革的 5 個維度」（太雜亂） | 「大學過渡期的自我概念下降機制」（單一焦點） |
| **濃縮** | 150–200 字，去蕪存菁 | 冗長段落，包含多個例子和子觀點 | 直擊核心，1–2 個例子最多 |
| **可複用** | 洞察應適用於多個場景 | 「某一特定班級的現象」 | 「準則參照框架轉變對自我評價的影響」 |
| **溯源** | 清楚標示來自哪篇論文或資料 | sources 欄位缺失 | sources 指向具體的摘要頁 |

---

## 使用指引

1. **新增論文時**：複製本模板結構，按 5-Pass 流程逐步完成分析
2. **Pass 5 卡片**：完成後應移動到 `wiki/cards/` 目錄，並在 `wiki/cards/index.md` 中註冊
3. **跨連結**：確保卡片和主 sources 頁面之間有雙向連結，形成知識網路
4. **版本控制**：每次更新時在 `last_updated` 欄位更新日期
